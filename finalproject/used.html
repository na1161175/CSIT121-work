Usage

Touch events are implemented by developers through the addition of event listeners on four main event types: the touchstart event fires when a finger touches the screen, touchmove fires when it moves with the finger still touching the screen, touchend fires when the finger lifts off, and touchcancel fires when something interrupts the touch. These can be added to specific elements or the entire document to handle user interactions on a touch screen. Some of the most useful information included in touch events includes touches, which show all active touches on screen, targetTouches, which show touches on a specific element, and changedTouches, which show which touches have changed since the last event. Each touch also includes coordinate properties such as pageX, pageY, clientX, and clientY, telling you exactly where the touch occurred.

Example 1: Gesture Detection

Based on the number of fingers touching the screen at the same time, applications may respond differently. The switch statement can be used by the touchstart handler to call different functions according to the number of active touches. Due to this, apps are able to support single or two-finger gestures, such as pinch to zoom and three-finger gestures for even more complex actions. 


          // Registers touch event handlers
someElement.addEventListener("touchstart", processTouchStart);
someElement.addEventListener("touchmove", processTouchMove);
someElement.addEventListener("touchend", processTouchEnd);

// touchstart handler
function processTouchStart(event) {
  // Use the event's data to call out to the appropriate gesture handlers
  switch (event.touches.length) {
    case 1:
      handleOneTouch(event);
      break;
    case 2:
      handleTwoTouches(event);
      break;
    case 3:
      handleThreeTouches(event);
      break;
    default:
      gestureNotSupported(event);
      break;
  }
}

Example 2:Swipe-to-Unlock Button

Most touch devices have the feature to unlock the device by swiping, which requires users to drag a button. The touchstart event captures the starting position when the finger touches the button. When the finger moves, the touchmove event compares the current finger position to the initial position and calculates how far the button must move. It then updates the button's position on screen. In touchend, the code checks to see if the button has reached the end of its track when the finger is lifted. If it does, the screen unlocks, and if it does not, it goes back to its starting point: 


          const button = document.getElementById("swipe-button");
const container = document.getElementById("swipe-container");
let startX = 0;

button.addEventListener("touchstart", (e) => {
  startX = e.touches[0].clientX;
});

button.addEventListener("touchmove", (e) => {
  const currentX = e.touches[0].clientX;
  const offset = Math.min(
    container.offsetWidth - button.offsetWidth,
    currentX - startX
  );
  button.style.transform = `translateX(${offset}px)`;
});

button.addEventListener("touchend", () => {
  const currentOffset =
    parseInt(button.style.transform.replace("translateX(", "")) || 0;

  if (currentOffset >= container.offsetWidth - button.offsetWidth - 10) {
    alert("Unlocked!");
  } else {
    button.style.transform = "translateX(0)";
  }
});
